Layouts.

Page chrome shapes. The named Layout variants (the outer wrapper, the SideNav slot, the TopNav slot) and the named LayoutGroup variants (the band recipes that hold Sections). This page is not about the column grid — that lives in grid.html.

01

The five-layer hierarchy

Every TransportNY surface nests through these layers. The Layout owns the chrome; the LayoutGroup owns a band; the Section owns one editable block; the Component is the actual rendering. 

Site
└── Layout                  ← every page is wrapped in one
    ├── (TopNav)             ← optional, configured per layout
    ├── (SideNav)            ← present on every TransportNY product surface
    └── childWrapper
        └── LayoutGroup      ← one per "band" (header / content / footer)
            └── LayoutGroup wrapper
                └── Section  ← one per content unit (the editable block)
                    └── Component (from the registry)
Layout owns
Page background, the outer wrapper, whether/how TopNav and SideNav render, the content area where LayoutGroups live. Three named styles ship: default · app · bare.
LayoutGroup owns
A band of content within a Layout. Sets the band's background (pane / pane-tint / topo / bluebar), the max-width container, the vertical padding, the gap between Sections inside.
02

Naming

For marketing pages
Layout default · LayoutGroup hero + content + footer.
For product pages
Layout app · LayoutGroup header + tone_bar (optional) + content.
For dense workbenches
Layout app · LayoutGroup workbench (full-bleed, edge-to-edge).
For auth flows
Layout bare · LayoutGroup auth.

05

LayoutGroup wrapper class reference

Every LayoutGroup renders as <wrapper1> → <wrapper2> → <wrapper3>. wrapper1 is the band (full-viewport-width, sets background and vertical padding). wrapper2 is the bounded inner container (max-width, horizontal alignment, internal padding). wrapper3 is rarely used. The class strings below are what theme.layoutGroup.styles[N] ships.

// rule
Product LayoutGroups use mr-auto, not mx-auto.

mx-auto centres content within the available width, which feels marketing-like on wide monitors — content drifts away from the sidebar's right edge. mr-auto pins content flush-left (with px-8 for breathing room) and consumes the remaining viewport on the right. That's how the handoff's <main className="ml-60"> behaves naturally and what every product page should match. Only auth stays centred — sign-in forms are the brand's one centred surface.

06

Section width — regular vs full-width groups

Inside any LayoutGroup, the sectionArray applies one of two layout classes to its grid container, chosen by the group.full_width flag (set per-group in the section-groups editor — Full Width: off or show). Source: src/dms/packages/dms/src/patterns/page/components/sections/sectionArray.jsx. 

// sectionArray.jsx · line ~161
<div className={`
    ${theme.container}
    ${theme.layouts[group.full_width === 'show' ? 'fullwidth' : 'centered']}
`}>
Regular · default
group.full_width: 'off' · layouts.centered
band wrapper2 · max-w-[1480px] mr-auto
↑ band sits flush-left; right side absorbs extra space

The default. Class: 'max-w-[1480px] mr-auto'. Sections lay out on the 12-col grid within the 1480-px cap. Use for every LayoutGroup except the workbench/canvas case.

Full-width · opt in
group.full_width: 'show' · layouts.fullwidth
band wrapper2 · w-full px-0
↑ band edge-to-edge; map / canvas content owns the width

Author opt-in via the section-groups editor toolbar (Full Width: show). Class: '' (empty — the wrapper's own constraints don't apply). Pair with the workbench LayoutGroup so the band also drops its max-width. Used for: the TMC segment map on map-21, dense canvases.

How the author toggles it

In the page editor's section-groups pane (patterns/page/pages/edit/editPane/sectionGroupsPane.jsx), each group exposes a "Full Width" menu item with two options: off (default) and show. Selecting show sets group.full_width = 'show' on the row, which sectionArray reads to pick between layouts.centered and layouts.fullwidth. There's no third option — it's binary.

A canonical TransportNY product page (e.g. map-21.html) is Layout 'app' → [header] · [tone_bar] · [content] · [workbench] · [footer]. Each band is a single LayoutGroup; each LayoutGroup hosts one or more Sections.

08

Section padding & the card gutter

A band's section grid is grid grid-cols-12 gap-0 — there is no grid gap. Instead, each section's padding is the gutter: it's rendered on the section wrapper, outside the card chrome box (the border / radius / background sit on an inner box inside the padding — see sectionArray.jsx · sectionChrome). Margins are never used (they fight grid/flex). The padding inside the card — text inset from the border — is the component's own concern (a lexical card uses theme.richtext.contentPadding, p-4).

Default gutter
defaultPaddingStep: "3" → p-3/side → 24px between cards = the mockups' gap-6
card
card

Leave padding unset on a card section and it inherits this default. Two neighbours each contribute p-3 → a 24px gutter, matching the gap-6 the freight-atlas + landing mockups use. This is the right call for almost every card — don't set per-section padding just to space cards.

Per-side override
padding: { bottom: "0" } · zero a shared edge to fuse a compound card
header section · padding.bottom 0
chart section · padding.top 0

A per-side object ({ top, right, bottom, left }) overrides individual sides against defaultPaddingStep; zeroing the shared edge fuses two sections into one card. A bare literal string ("p-6") overrides all sides — handy occasionally, but a page-wide habit of p-4/5/6 over-inflates the gutters past the design (the freight-atlas home first shipped that way; the fix was to drop the overrides).

// Padding steps

Steps map to Tailwind per side via theme.pages.sectionArray.paddings: "0" (flush) · "2" (16px gutter) · "3" (24px — default) · "4" (32px) · "6" · "8". The author sets these per-side in the section toolbar (Settings → Layout → Padding). Change the default only to re-space the whole brand; to nudge one page, override that page's sections.

// Equal-height rows

For a row of more than one card (KPI strip, tile/feature grid, a tall card beside a short one), set each card section's height: 'fill' so they stretch to the row's height and read as an even set instead of ragged, content-shrunk boxes. Leave height unset (auto) — the default — for a full-width section or a card that's alone in its row. (Author UI: section toolbar → Settings → Layout → Height.)
