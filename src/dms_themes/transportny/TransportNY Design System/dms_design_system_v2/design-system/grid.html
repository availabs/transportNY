Grid.

The page-content column grid every Section sits on inside a LayoutGroup. TransportNY ships a 12-column grid (the brand's data container is 1480 px wide). This page is the column-span vocabulary authors choose from.

01

The grid spec

From theme.pages.sectionArray.styles[0] — the sectionArray override that every LayoutGroup uses for its inner column grid. All values are wholesale-replaced (_replace: ['sizes']) so legacy 6-col keys from the codebase default don't leak through.

Key Value Why
gridSize12Twelve columns gives authors 8.3% increments — fine enough for 5/12, 7/12, 8/12 asymmetric splits.
defaultSize"12"A new section dropped via "+ Add" starts full-width. Authors set a narrower span explicitly when needed.
container"w-full grid grid-cols-12 gap-0"gap-0 — the gutter is per-section padding, not grid gap, so a section can zero one edge and sit flush with its neighbour (compound cards). No margins (they fight grid/flex).
defaultPaddingStep"3"The gutter step applied to every side by default (≈24 px between sections). Zero a side to compose flush.
paddings{ top/right/bottom/left: { 0,2,3,4,6,8 } }Curated per-side gutter steps (literal `pt-3` … so Tailwind generates them). Drives the Layout → Padding picker (All + per-side).
borderSides{ top/right/bottom/left }Per-side border lines (one brand hairline). Rendered on the section's inner box so the padding is a true gutter. Legacy `border` preset keys still resolve (BC).
radiusCorners / backgrounds{ tl/tr/bl/br } · { none/white/tint }Per-corner radius + inner-box background. Together with borderSides these make any section a card; coordinate them across two sections to fuse a compound card.
layouts.centered"max-w-[1480px] mx-auto"The brand "data" container width. Overrides codebase default (1020 px) which would render content narrow.
layouts.fullwidth""For the rare full-bleed band — the map workbench.
sectionPadding"p-3"Fallback gutter when `defaultPaddingStep` is unset. Content padding inside a card is the component's own concern, not the section.
sizes{ "1" … "12" }Numeric keys — see vocabulary table below. Replaces codebase's "1" / "1/2" / "1/3" / "2/3".
rowspans{ "1": "row-span-1", "2": "row-span-2", "3": "row-span-3" }Row-spans used sparingly. Most sections sit on a single row.

Row-span vocabulary

Row-spans are used sparingly — most TransportNY pages stack sections horizontally and don't reach for vertical-span. The common case: a tall sidebar (map / inspector) next to two stacked sections. rowspans: { "2": "row-span-2" }.

06

Picker rules

Do
  • Default to full-row. If the section doesn't need to share space, leave it at 12.
  • Favor symmetric splits. 6+6, 4+4+4, 3+3+3+3 are easier to scan than 5+7 or 7+5.
  • Use 8+4 for "primary content + helper rail" — the most common asymmetric pattern.
  • Reserve row-spans for tall companions. A map next to two stacked panels, an inspector beside a leaderboard.
Don't
  • Pin a section to size: "1" unless you actually want a 1/12 sliver. New sections default to 12 — that's intentional.
  • Use asymmetric splits like 5+7 without a reason. If the data justifies it, fine; otherwise pick 6+6 or 4+8.
  • Nest grids inside a Section. The Card's cellsGridSize handles intra-section layout — see components.html.
  • Reach for layouts.fullwidth outside the map workbench. The 1480-px container is correct for almost every band.
